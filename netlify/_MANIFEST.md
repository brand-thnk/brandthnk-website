# netlify — _MANIFEST.md

## Purpose
Netlify deployment configuration and serverless functions. Controls build process, redirects, and server-side logic for the insights-tool site.

## When to Use
- Modifying deployment settings or build configuration
- Adding or updating serverless functions
- Configuring redirects or rewrites
- Changing build command or environment variables
- Maintaining API proxies or backend logic

## Key Files

| File | Purpose |
|------|---------|
| BrandThnk/Tools/insights-tool/website/netlify/netlify.toml | Master deployment configuration (build, redirects, functions config) |
| BrandThnk/Tools/insights-tool/website/netlify/functions/anthropic-proxy.js | API proxy for Anthropic (keeps API key server-side) |
| BrandThnk/Tools/insights-tool/website/netlify/functions/sign-contract.js | Contract signing logic and workflow |
| BrandThnk/Tools/insights-tool/website/netlify/functions/therapy-capture.js | Session capture webhook for feedback/therapy data |

## netlify.toml Configuration
- Build command: esbuild bundler
- Redirect rules: All old paths (e.g., /do-portal/*, /hc3-portal/*) 301 redirect to /client-portals/ equivalents
- Functions routing: Maps function names to HTTP endpoints

## Serverless Functions
All functions are Node.js-based:

- **anthropic-proxy.js**: Proxies requests to Anthropic API
  - Keeps API key on server (never exposed to client)
  - Used by insights tool for AI-powered analysis

- **sign-contract.js**: Contract signing and workflow
  - Handles contract acceptance
  - Manages signature data

- **therapy-capture.js**: Captures user feedback/therapy data
  - Webhook endpoint for feedback submission
  - Stores session/interaction data

## Dependencies
- Git push to main branch triggers automatic Netlify deploy
- Build artifacts generated by esbuild
- Environment variables (API keys) managed in Netlify dashboard
- Functions execute on-demand when called from client code

## Important Notes
- Never commit API keys or secrets to repo (use Netlify env vars)
- Test redirects after modifying BrandThnk/Tools/insights-tool/website/netlify/netlify.toml
- Functions must return valid JSON or HTTP responses
- Keep function code minimal and focused
